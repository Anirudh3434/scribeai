info

macOS may show a security warning on first launch

Because ScribeAI is distributed outside the Mac App Store, macOS Gatekeeper will show a warning the very first time you open it. This is normal for independent apps. There are two easy ways to bypass it:

mouse Option A — Right-click method (easiest)

  1. 1.Go to Applications in Finder
  2. 2.Right-click ScribeAI → choose Open
  3. 3.Click Open in the dialog that appears
  4. ✓Done — opens normally every time after this

terminal Option B — Terminal command

Paste this once in Terminal after dragging to Applications:

xattr -dr com.apple.quarantine /Applications/ScribeAI.app

Common Questions

Everything you need to know

How does the global macOS shortcut work? expand_more
ScribeAI runs as a lightweight accessibility agent. When you highlight text and press the hotkey, it captures the selection via macOS APIs, processes it locally using the embedded LLM, and offers to replace it or copy the result to your clipboard.
What are the system requirements for offline WebGPU models? expand_more
For the best experience, we recommend an Apple Silicon Mac (M1/M2/M3) with at least 8GB of RAM. The WebGPU implementation works in Chrome 113+ and Safari 17+. If your hardware is older, you can toggle on "API Mode" to use cloud models.
Is ScribeAI completely free? expand_more
Yes, the core ScribeAI client is free and open-source. Since the AI runs on your own hardware, there are no server costs for us to pass on to you. If you choose to use third-party APIs (like OpenAI), you pay only for your own usage directly to them.
Why doesn't the AI change proper names or spelling of names? expand_more
We've implemented a custom "Identity Lock" layer. Before the AI processes your text, we index proper nouns and names. After processing, we run a verification pass to ensure the AI hasn't accidentally regularized unique spellings or changed cultural names.
Why does the downloaded DMG file group under the year 1984 in my Downloads folder? expand_more
This is a harmless macOS / Safari quirk. When downloading files via GitHub Releases and S3 redirects, Safari occasionally does not receive a standard Last-Modified date header. If no date is found, macOS Safari defaults the file creation timestamp to **January 24, 1984**—the launch date of the original Apple Macintosh computer! The file is completely intact and safe to open.
